Actions recommandees

Ordre conseille pour corriger les risques sans perturber le fonctionnement valide.

  1. Passer le logging Spring Security hors DEBUG en production afin de ne plus exposer IP, contexte session et URLs OAuth.
  2. Valider la configuration proxy Keycloak et n'accepter qu'un type d'en-tete transmis par le proxy de confiance ; remplacer l'option proxy obsolete.
  3. Activer ou justifier la transaction recovery pour le datasource XA de Keycloak.
  4. Mettre a niveau Flyway pour une compatibilite PostgreSQL 17 testee, puis retirer la configuration Hibernate depreciee et statuer sur open-in-view.
  5. Regler l'execution Supervisor/root explicitement et evaluer les avertissements Infinispan/JGroups selon le mode de deploiement mono-noeud du Space.
